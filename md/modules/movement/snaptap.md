## SnapTap

Prioritizes the last pressed movement key. When you hold two opposite movement keys at the same time (for example left and right, or forward and backward), the most recently pressed key wins instead of the inputs cancelling each other out. This reproduces the "SOCD" / null-movement behaviour used for instant counter-strafing.

**Category:** Movement
**Aliases:** NullMove, SOCD
**Enabled by default:** No

### Settings

This module has no configurable settings.

### Frequently Asked Questions

**Q: Does this work while an inventory or chat screen is open?**
A: Only if [InventoryMove](/docs/modules/movement/inventorymove) is also enabled. Otherwise the override pauses while a screen is open.
